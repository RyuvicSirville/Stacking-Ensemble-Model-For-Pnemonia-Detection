# Stacking Ensemble Model For Pneumonia Detection

## 📋 Project Overview

This project implements a **Stacking Ensemble approach** to detect pneumonia from chest X-ray images. The ensemble combines predictions from four deep learning base models to achieve robust and accurate pneumonia classification. The primary goal is to build a reliable medical imaging classifier by leveraging the strengths of multiple models.

### Key Features

- 🎯 **4 Base Models**: EfficientNetB3, DenseNet121, ResNet50V2, and VGG16 (with VDE enhancement)
- 🔗 **Stacking Ensemble**: Meta-learner approach combining all base model predictions
- 🖼️ **Medical Imaging**: Trained on chest X-ray datasets for pneumonia classification
- 📊 **Enhanced Performance**: Logistic Regression as meta-learner for optimal prediction aggregation
- ⚡ **Deep Learning**: TensorFlow/Keras implementation with GPU support

---

## 🏗️ Project Architecture

### Base Models (4)

1. **EfficientNetB3** (`EfficientNetB3.ipynb`)
   - Efficient CNN architecture optimized for performance and accuracy
   - Pre-trained on ImageNet
   - Input size: 224×224 pixels
   - Includes data augmentation and early stopping

2. **DenseNet121** (`modelDense.ipynb`)
   - Densely connected convolutional network
   - Dense skip connections for better gradient flow
   - Pre-trained ImageNet weights
   - Comprehensive evaluation metrics

3. **ResNet50V2** (`resnet50v2.ipynb`)
   - Residual Network with improved training stability
   - Version 2 architecture with better optimization
   - Pre-trained weights initialization
   - Batch normalization and regularization

4. **VGG16 with VDE** (`VGG16_VDE.ipynb`)
   - Visual Domain Enhancement (VDE) variant
   - Feature masking for domain-specific improvements
   - Lightweight yet powerful feature extraction
   - Enhanced generalization capability

### Stacking Ensemble (`StackingEnsemble.ipynb`)

The stacking ensemble combines all four base models:

1. Each base model generates predictions on the validation/test set
2. These predictions are stacked as features
3. A Logistic Regression meta-learner is trained on these features
4. Final predictions are generated by the meta-learner

**Benefits of Stacking:**

- Combines strengths of diverse architectures
- Reduces overfitting from any single model
- Improves robustness and generalization
- Leverages complementary model information

## 🎯 Stacking Ensemble Workflow

### Phase 1: Base Model Training

```
Dataset → [Base Model 1] → Predictions
       → [Base Model 2] → Predictions
       → [Base Model 3] → Predictions
       → [Base Model 4] → Predictions
```

### Phase 2: Meta-Learner Training

```
Base Model Predictions → Stack Features
                      → Logistic Regression (Meta-Learner)
                      → Final Classification
```

### Advantages

1. **Diversity**: Different architectures capture different features
2. **Robustness**: Less susceptible to any single model's biases
3. **Complementarity**: Some models excel on certain patterns
4. **Improved Accuracy**: Meta-learner optimally weighs model outputs
5. **Reduced Overfitting**: Ensemble generalizes better to new data
